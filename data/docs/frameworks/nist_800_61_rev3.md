# NIST SP 800-61 Revision 3: Incident Response Recommendations (2025)

## Overview
NIST SP 800-61 Revision 3 (released April 2025) replaces the traditional 4-phase lifecycle with a framework aligned with the **NIST Cybersecurity Framework (CSF) 2.0**. It emphasizes risk management integration and dynamic guidance.

## Core Functions (CSF 2.0 Alignment)
1. **Govern (GV)**: Establish incident response policy, roles, and responsibilities.
2. **Identify (ID)**: Understand the assets and systems that need protection.
3. **Protect (PR)**: Implement safeguards to ensure delivery of critical services.
4. **Detect (DE)**: Monitor systems for potential security events.
5. **Respond (RS)**: Take action once a security event is detected.
6. **Recover (RC)**: Restore capabilities or services that were impaired.

## Key Changes from Revision 2
- **Lifecycle Shift**: Moves from a standalone "Detection & Analysis" focus to an integrated "Risk Management" focus.
- **Dynamic Guidance**: Links to live NIST resources for rapidly changing technical details.
- **Resource Prioritization**: Focus on prioritizing incidents based on business impact rather than just technical severity.

## Recommendations for SOC Analysts
- Alignment with organizational risk tolerance is paramount.
- Use the **Respond (RS)** function for active containment and eradication:
    - RS.MA: Incident management.
    - RS.AN: Analysis of the incident.
    - RS.MI: Mitigation of the incident.
    - RS.CO: Communication with stakeholders.
